# ROS 2 کا تعارف - روبوٹک کا عصباتی نظام

## باب کا جائزہ

روبوٹکس کی دلچسپ دنیا میں خوش آمدید! ROS 2 (روبوٹ آپریٹنگ سسٹم 2) کے اس باب میں، ہم ROS 2 - کو درساتے ہیں جو روبوٹکس ایپلی کیشنز کو تقویت دینے والے مڈل ویئر کا کام کرتا ہے. ROS 2 کو اس طرح سوچیں جیسے ایک روبوٹ کا "عصبی نظام" - مختلف اقسام کے کمپونینٹس کو مواصلت، اور ایک دوسرے کے ساتھ کام کرنے کی سہولت فراہم کرتا ہے. اس باب کے اختتام تک، آپ ROS 2 کے بنیادی تصورات کو سمجھیں گے اور روبوٹکس ایپلی کیشنز تعمیر کرنے کے لیے تیار ہو جائیں گے.

## ROS 2 کیا ہے؟

ROS 2 روبوٹکس ایپلی کیشنز تعمیر کرنے کے لیے ایک اوپن سورس فریم ورک ہے. ROS 2 کو ایک روبوٹ کی "دماغ" کے طور پر سوچیں - ہر ایک ذمہ داری ایک مخصوص کام کے لیے ہوتی ہے۔ ROS 2 مختلف اقسام کے کمپونینٹس کو مواصلت کرنے اور ایک دوسرے کے ساتھ کام کرنے کی سہولت فراہم کرتا ہے.

### ROS 1 سے ROS 2 کی ترقی

ROS 2 نے ROS 1 کے جانشین کے طور پر ترقی کی تاکہ ریل ورلڈ ایپلی کیشنز میں روبوٹکس سسٹمز کو منتقل کرنے میں مشکلات کو حل کیا جاسکے. ROS 1 کی ایجاد تحقیق اور ترقی کے لیے بہتر تھی، لیکن حقیقی وقت کی کارکردگی، سیکورٹی، متعدد روبوٹ سسٹمز، خدمات کی معیار، اور زندگی کے دوران کے انتظام کے شعبوں میں اس کی حدود موجود تھیں.

### ROS 2 کی کلیدی خصوصیات

- **پلیٹ فارم کی آزادی**: لینکس، میک، ونڈوز، اور این بیڈڈ سسٹمز پر کام کرتا ہے
- **زبان کی آزادی**: پائی تھون، سی ++، جاوا، اور دیگر زبانوں کو سپورٹ کرتا ہے
- **تقسیم شدہ کمپیوٹنگ**: متعدد کمپیوٹرز کو ایک ساتھ بے سمتگی سے کام کرنے کی اجازت دیتا ہے
- **متنوع اکوسسٹم**: ادراک، نیوی گیشن، کنٹرول، اور مزید کے لیے ہزاروں پیکجز
- **صنعت کا استعمال**: ایمیزون، گوگل، ٹویوٹا اور ایئربس جیسی کمپنیوں کی طرف سے استعمال کیا جاتا ہے

## ROS 2 کے بنیادی تصورات

ROS 2 کے بنیادی تصورات کو سمجھنا اہم ہے قبل ایپلی کیشن میں گہرائی سے جانا رکھنا. آئیے بنیادی ڈھانچہ جات کا جائزہ لیتے ہیں.

### نوڈز

ایک **نوڈ** ایک عمل ہے جو کمپیو ٹیشن کرتا ہے. ROS 2 میں، نوڈز کمپیو ٹیشن کی بنیادی اکائیاں ہیں. نوڈز کو ایک روبوٹ کے "دماغ" کے الگ الگ ایجنٹ کے طور پر سوچیں - ہر ایک ذمہ داری ایک مخصوص کام کے لیے ہوتی ہے.

نوڈز کر سکتے ہیں:
- ٹاپکس کو پبلش کرنا
- ٹاپکس کو سبسکرائب کرنا
- سروسز فراہم کرنا
- سروسز کال کرنا
- ایکشنز انجام دینا

ایک موبائل روبوٹ میں، آپ کے پاس موتور کنٹرول، سینسر پروسیسنگ، نیوی گیشن یو ایچ ڈی، اور صارف انٹرفیس کے لیے نوڈز ہوسکتے ہیں، جو سب ROS 2 کے ذریعے مواصلت کرتے ہیں.

### ٹاپکس

ایک **ٹاپک** ایک نامزد چینل ہے جس کے ذریعے ROS نوڈز پیغامات بھیجتے اور حاصل کرتے ہیں. ٹاپکس ایک پبلش-سبسکرائب مواصلت ماڈل کو لاگو کرتے ہیں جہاں پبلشرز ٹاپک میں ڈیٹا بھیجتے ہیں اور سبسکرائبرز ٹاپک سے ڈیٹا حاصل کرتے ہیں. یہ نوڈز کے درمیان غیر مترتب مواصلت کی اجازت دیتا ہے.

مثال کے طور پر، ایک کیمرہ نوڈ `/کیمرہ/امیج_راو` نامی ٹاپک کو سینسر ڈیٹا بھیج سکتی ہے، اور متعدد نوڈز اس ڈیٹا کو حاصل کرنے کے لیے سبسکرائب کر سکتے ہیں - شاید ایک تصویری پروسیسنگ کے لیے، ایک دوسری ویژولائزیشن کے لیے، اور تیسری اسٹوریج کے لیے.

### پبلشرز اور سبسکرائبرز

**پبلشرز** ٹاپک میں پیغامات بھیجتے ہیں، جبکہ **سبسکرائبرز** ٹاپک سے پیغامات حاصل کرتے ہیں. یہ ROS 2 میں مواصلت کا سب سے عام نمونہ ہے.

- پبلشرز اور سبسکرائبرز الگ الگ ہیں: ایک پبلشر یہ نہیں جانتا کہ کون سب سبسکرائب کر رہا ہے
- متعدد سبسکرائبرز ایک ہی ٹاپک کو سن سکتے ہیں
- متعدد پبلشرز ایک ہی ٹاپک کو پبلش کرسکتے ہیں (اگرچہ یہ کم عام ہے)

### سروسز

سروسز ہم وقت درخواست-جواب مواصلت فراہم کرتی ہیں، جس کی مثال ایک فعال کال کی طرح ہے. ایک کلائنٹ سروس کو ایک درخواست بھیجتا ہے اور جواب کا انتظار کرتا ہے.

سروسز لمبے وقت تک کے کاموں کے لیے اچھی ہیں جن میں جواب کی ضرورت ہوتی ہے، جیسے:
- ایک نقشہ محفوظ کرنا
- نیوی گیشن سسٹم میں ایک ویپوائنٹ شامل کرنا
- موجودہ روبوٹ کا اصول حاصل کرنا

### ایکشنز

ایکشنز طویل وقت تک کے کاموں کے لیے ایک اعلی شکل ہیں جن کو فیڈ بیک اور منسوخ کرنے کی صلاحیت کی ضرورت ہوتی ہے. وہ گو کو کے مقصد کے کاموں جیسے نیوی گیشن تک پہنچنا، ایک روبوٹ کے بازو کو حرکت دینا، یا ایک پیچیدہ مینیپولیشن کا کام انجام دینے کے لیے مفید ہیں.

## ROS 2 کے ساتھ کام کرنا

### ROS 2 انسٹال کرنا

ROS 2 کا سب سے عام انتشار "ہمبل ہاکسبل" ہے، جو ایک LTS (طویل المدت تعاون) انتشار ہے. اس کو انسٹال کرنے کے لیے:

Ubuntu 22.04 کے لیے:
```bash
sudo apt update
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update
sudo apt install curl gnupg2 lsb-release
curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key | sudo apt-key add -
sudo sh -c 'echo "deb http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" > /etc/apt/sources.list.d/ros2-latest.list'
sudo apt update
sudo apt install ros-humble-desktop
sudo apt install python3-colcon-common-extensions
```

اپنے سیٹ اپ کو سورس کرنا مت بھولیں:
```bash
source /opt/ros/humble/setup.bash
```

### اپنا پہلا ROS 2 ورک سپیس بنانا

آئیے ایک ورک سپیس بنائیں جہاں ہم اپنے ROS 2 پیکجز تعمیر کریں:

```bash
# ایک نیا ورک سپیس بنائیں
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws

# ورک سپیس کو تعمیر کریں (اگرچہ یہ ابھی خالی ہے)
colcon build

# ورک سپیس کو سورس کریں تاکہ اندر کے پیکجز کا استعمال کیا جا سکے
source install/setup.bash
```

## عملی مثال: ایک سادہ پبلشر بنانا

آئیے ایک سادہ ROS 2 پیکیج بنائیں اور ایک پبلشر اور سبسکرائبر لکھیں.

سب سے پہلے، ایک نیا پیکج بنائیں:
```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python py_pubsub
```

اب ایک پبلشر نوڈ بناو. `~/ros2_ws/src/py_pubsub/py_pubsub/publisher_member_function.py` میں:

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class MinimalPublisher(Node):

    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello Robot %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)
        self.i += 1


def main(args=None):
    rclpy.init(args=args)

    minimal_publisher = MinimalPublisher()

    rclpy.spin(minimal_publisher)

    # Destroy the node explicitly
    minimal_publisher.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

اور اب ایک سبسکرائبر `~/ros2_ws/src/py_pubsub/py_pubsub/subscriber_member_function.py` میں:

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class MinimalSubscriber(Node):

    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info('I heard: "%s"' % msg.data)


def main(args=None):
    rclpy.init(args=args)

    minimal_subscriber = MinimalSubscriber()

    rclpy.spin(minimal_subscriber)

    # Destroy the node explicitly
    minimal_subscriber.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

ان کو چلانے کے لیے:
ٹرمنل 1:
```bash
cd ~/ros2_ws
source install/setup.bash
ros2 run py_pubsub talker
```

ٹرمنل 2:
```bash
cd ~/ros2_ws
source install/setup.bash
ros2 run py_pubsub listener
```

## خدمات کا عملی استعمال

اب آئیے ایک خدمات بنانا دیکھتے ہیں. `~/ros2_ws/src/py_pubsub/py_pubsub/add_three_ints.srv` میں خدمات کی تعریف فائل بنائیں:

```
int64 a
int64 b
int64 c
---
int64 sum
```

یہ خدمت تین عددی متغیروں کو لیتی ہے اور ان کا مجموعہ لوٹاتی ہے. خدمات کا سرور بنائیں:

```python
import rclpy
from rclpy.node import Node

from example_interfaces.srv import AddTwoInts


class MinimalService(Node):

    def __init__(self):
        super().__init__('minimal_service')
        self.srv = self.create_service(AddTwoInts, 'add_two_ints', self.add_two_ints_callback)

    def add_two_ints_callback(self, request, response):
        response.sum = request.a + request.b
        self.get_logger().info('Incoming request\na: %d b: %d' % (request.a, request.b))

        return response


def main(args=None):
    rclpy.init(args=args)

    minimal_service = MinimalService()

    rclpy.spin(minimal_service)

    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## معیار کے سروس (QoS) ترتیبات

ROS 2 میں متعارف کرائے گئے معیار کے سروس (QoS) ترتیبات میں سے ایک طاقتور خصوصیت ہے. QoS اجازت دیتا ہے کہ آپ مواصلت کی خصوصیات کو فائن ٹیون کریں، جو حقیقی وقت اور محفوظ ایپلی کیشنز کے لیے اہم ہے.

عام QoS ترتیبات میں شامل ہیں:
- قابل اعتماد: قابل اعتماد (یقینی ترسیل) یا بہترین کوشش (ترسیل کی ضمانت نہیں)
- دائمی: متزلزل (نئے سبسکرائبرز کو ماضی کے پیغامات نہیں ملتے) یا عارضی مقامی (نئے سبسکرائبرز کو آخری معلوم قدر ملتی ہے)
- تاریخ: تاریخ دیکھنے کے لیے کتنے پیغامات کو برقرار رکھنا ہے

## مشقیں

1. **پبلشر کو موڈیفائ کریں**: پبلشر کو ایڈجسٹ کریں تاکہ وہ سادہ سٹرنگس کے بجائے سینسر ڈیٹا بھیجے. شاید 0.1 میٹر سے 10.0 میٹر تک کی دوری کے سینسر کی شبیہہ بنائیں.

2. **ایک ڈیٹا لاگر بنائیں**: ایک نیا نوڈ بنائیں جو دوری سینسر ٹاپک کو سبسکرائب کرے اور ڈیٹا کو ٹائم اسٹیمپ کے ساتھ ایک فائل میں لاگ کرے.

3. **خدمت کو بہتر بنائیں**: ایک خدمات بنائیں جو ایک عدد لیتی ہے اور یہ جانچتی ہے کہ کیا یہ عدد مفرد (Prime) ہے. خدمات سرور اور کلائنٹ دونوں کو لاگو کریں.

## مینی-پروجیکٹ: سادہ درجہ حرارت مانیٹر

ایک نظام تعمیر کریں جو کہ:
1. ایک درجہ حرارت سیمولیٹر جو 15°C اور 35°C کے درمیان کچھ عدد بھیجے
2. ایک مانیٹر نوڈ جو درجہ حرارت ٹاپک کو سبسکرائب کرے اور 30°C سے زائد ہونے پر انتباہ چھاپے
3. ایک ڈیٹا لاگر جو تمام درجہ حرارت کے پڑھاؤ کو ٹائم اسٹیمپ کے ساتھ ریکارڈ کرے
4. ایک تاریخی ڈیٹا خدمات جو، جب کال کی جاۓ، آخری 10 ریڈنگز پر مبنی اوسط درجہ حرارت لوٹاۓ

## خلاصہ

اس باب میں، ہم نے ROS 2 کے بنیادی تصورات پر گفتگو کی:
- نوڈز کے طور پر اساسی کمپیو ٹیشنل یونٹس
- ٹاپکس کے لیے غیر مترتب پبلش-سبسکرائب مواصلت
- خدمات کے لیے ہم وقت درخواست-جواب مواصلت
- سادہ پبلشر اور سبسکرائبر نوڈز کیسے تعمیر کریں
- معیار کے سروس (QoS) ترتیبات کے لیے موثر مواصلت کے رویے کو کیسے فائن ٹیون کریں

یہ علم آپ کو پیچیدہ روبوٹکس سسٹمز تعمیر کرنے کے لیے بنیاد فراہم کرے گا. اگلے باب میں، ہم ROS 2 اوزار، پیرامیٹر مینجمنٹ، اور زیادہ اعلیٰ مواصلت کے نمونوں میں گہرائی سے جائیں گے.

## جائزہ سوالات

1. ROS 2 میں پبلشر-سبسکرائبر نمونہ اور خدمات نمونہ میں کیا فرق ہے؟
2. ROS 2 معماری میں DDS کا کیا کردار ہے؟
3. ROS 2 میں آپ کس وقت خدمات کا مقابلہ کریں گے اور ایکشنز کا مقابلہ کریں گے؟
4. معیار کے سروس (QoS) کی ترتیبات "قابل اعتماد" کیا ضمانت دیتے ہیں؟

## عام چیلنجز اور حل

ROS 2 میں نئے آنے والے اکثر درج ذیل چیلنجوں کو سامنا کرتے ہیں:
- نوڈ-ٹاپک کے تعلق کو سمجھنا
- ورک سپیس ماحولات کا نظم کرنا
- مواصلت کے مسائل کو ڈیبگ کرنا

یقین کریں کہ ROS 2 کمانڈز چلاۓ سے پہلے اپنے ورک سپیس کو سورس کریں، اور ڈیبگنگ کے لیے `ros2 topic list` اور `ros2 node list` جیسے اوزار استعمال کریں.

اب آپ ROS 2 کے مزید پیچیدہ پہلوؤں کا استعمال شروع کرنے کے لیے تیار ہیں اور مزید ترقی یافتہ روبوٹکس ایپلی کیشنز تعمیر کر سکتے ہیں!
